# 部品発注・管理・廃棄ワークフローシステム (PLMS)
このプロジェクトは、私たちだけのカスタムウェディング招待状を作成するために開発されたウェブアプリケーションです。
ポートフォリオとしても活用できるよう、プロジェクトの背景や技術的な工夫点を以下にまとめました。

## 1. プロジェクト概要
本プロジェクトは、製造現場やメンテナンス業務における部品の**「発注・受入・在庫管理・廃棄」**までのライフサイクルを一元管理するMVC構造の Web システムです。
特に、既存のレガシーシステムの処理を、Java 17 と Spring Boot を用いてオブジェクト指向設計へと再構築（モダナイゼーション）することに主眼を置いています。

## 2. 技術スタック (Tech Stack)

### Backend
- **Java 17**: Record, Stream API, Optional などの現代的な機能を活用
- **Spring Boot 3.x**: 高い生産性とメンテナンス性の確保
- **Spring Data JPA**: Hibernate を利用したデータアクセス層の抽象化
- **JUnit 5**: 業務ロジックの正当性を保証するための単体テスト

### Frontend
- **Thymeleaf**: サーバーサイドレンダリングによる堅牢な UI 実装
- **Tailwind CSS**: 迅速かつ一貫性のあるレスポンシブデザイン

### Database & Infrastructure
- **ThymePostgreSQL (Supabase)**: 外部接続を考慮したリレーショナルデータベース
- **Railway**: クラウド環境へのデプロイと環境変数管理


## 3. システムアーキテクチャ
本システムは、保守性を高めるために厳格なレイヤードアーキテクチャを採用しています。
1. **Controller**: リクエストの受付とバリデーション
2. **Service**: ビジネスロジックの実装
3. **Repository**: データアクセスの抽象化
4. **Entity**: ビジネス実体とデータ構造の定義

## 4. コアロジック：部品ライフサイクル管理

部品の状態（Status）をEnumで定義し、不整合な状態遷移を防ぐ設計を行いました。
- **発注**: 在庫不足時の補充申請
- **入庫**: 納品後の検収および在庫反映（トランザクション管理）
- **廃棄**: 劣化・不良品の廃棄承認プロセス

## 5. 技術的な注力ポイント
1. レガシーロジックの再構築
  COBOLのIF段組による複雑な判定を、JavaのOptionalやStreamを用いて、可読性と保守性の高いコードへとリファクタリングしました。
2. データ整合性の確保
  部品の発注から在庫反映まで、複数のテーブルに跨る更新処理において、Springのトランザクション管理を活用し、データの不整合（Atomicity）を徹底的に防止しました。
3. 環境依存情報の分離
  DB接続情報やAPIキーなどの機密情報をVercelの環境変数（Environment Variables）で管理し、GitHub上への情報露出を防ぐセキュアなデプロイを実現しました。
