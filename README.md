# 部品発注・管理・廃棄ワークフローシステム (PLMS)

## 1. プロジェクト概要
本プロジェクトは、製造現場やメンテナンス業務における部品の「発注・入庫・廃棄」のライフサイクルと在庫管理と計上（清算）を一元管理する本格的なMVC構造のWebシステムを開発してみました。
既存の現場でよく見たレガシーシステムの処理を、Java17とSpring Bootを用いてオブジェクト指向設計へと再構築し、データ整合性とスケーラビリティを確保しています。

## 2. 技術スタック (Tech Stack)

### Backend
- **Java 17**: Record, Stream API, Optional などの現代的な機能を活用
- **Spring Boot 3.x**: 高い生産性とメンテナンス性の確保
- **Spring Data JPA**: Hibernate を利用したデータアクセス層の抽象化
- **OpenPDF (iText)**: サーバーサイドでの動的なPDF帳票（精算書）生成とZIP圧縮
- **Gradle**: 高速化されたビルドと依存関係のスマートな管理

### Frontend
- **Thymeleaf**: サーバーサイドレンダリングによる堅牢で動的な UI 実装
- **Tailwind CSS**: ユーティリティファーストによる直感的でモダンなレスポンシブデザイン
- **Vanilla JavaScript**: モーダル制御およびDOM操作（軽量化のため外部ライブラリ非依存）

### Database & Infrastructure
- **PostgreSQL**: データ一貫性を担保する本格的なリレーショナルデータベース
- **Docker**: アプリケーション環境の完全なコンテナ化とポータビリティの向上
- **Railway**: GitHubフックとDockerfileを用いたクラウド環境への継続的デプロイ (CI/CD)

## 3. コア機能 (Core Features)

1. **部品・取引先マスター管理**
   部品や取引先のマスタ情報を管理、修正。
2. **発注管理**
   部品と取引先を指定して新規発注や発注内容の検索・修正。
3. **入庫管理**
   バーコード/QRリーダーでの入力を想定した入庫処理。在庫の即時加算と、トランザクションによる入庫履歴の確実な記録。
4. **在庫・廃棄管理**
   リアルタイムな在庫追跡。廃棄処理と理由のロギング。
5. **計上および精算書自動生成**
   指定期間内の入庫履歴を取引先別に集計。PDF精算書をバックエンドで動的に描画し、一括でZIPファイルとしてダウンロード可能。

## 4. 技術的な注力ポイント

1. **セキュアで堅牢なPDFエンジン設計**
   Spring BootのFAT JAR実行環境における静的ファイル参照問題を解決するため、TTFフォントリソースを起動時に一時領域へ物理・展開してからPDF描画エンジンへ渡すことで、環境非依存の安定稼働を実現。
2. **動的でユーザーフレンドリーなUI制御**
   発注ステータスや入庫数量などの条件付きロジックを Thymeleaf 上で計算し、ユーザーが誤った操作を行えないように編集ボタンや入力領域を動的に制御。
3. **環境依存情報の分離**
   DBの接続情報やデプロイ設定などの機密情報を環境変数へと分離し、GitHub上への情報露出を防ぐセキュアな設計。
