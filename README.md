# 部品発注・管理・廃棄ワークフローシステム (PLMS)

## 1. プロジェクト概要
本プロジェクトは、製造現場やメンテナンス業務における部品の**「発注・受入・在庫管理・廃棄・計上(精算)」**までのライフサイクルを一元管理する本格的なMVC構造の Web システムです。
既存のレガシーシステムの処理を、Java 17 と Spring Boot を用いてオブジェクト指向設計へと再構築（モダナイゼーション）し、データ整合性とスケーラビリティを確保しています。

## 2. 技術スタック (Tech Stack)

### Backend
- **Java 17**: Record, Stream API, Optional などの現代的な機能を活用
- **Spring Boot 3.x**: 高い生産性とメンテナンス性の確保
- **Spring Data JPA**: Hibernate を利用したデータアクセス層の抽象化
- **OpenPDF (iText)**: サーバーサイドでの動的なPDF帳票（精算書）生成とZIP圧縮
- **Gradle**: 高速化されたビルドと依存関係のスマートな管理

### Frontend
- **Thymeleaf**: サーバーサイドレンダリングによる堅牢で動的な UI 実装
- **Tailwind CSS**: ユーティリティファーストによる直感的でモダンなレスポンシブデザイン
- **Vanilla JavaScript**: モーダル制御およびDOM操作（軽量化のため外部ライブラリ非依存）

### Database & Infrastructure
- **PostgreSQL**: データ一貫性を担保する本格的なリレーショナルデータベース
- **Docker**: アプリケーション環境の完全なコンテナ化とポータビリティの向上
- **Railway**: GitHubフックとDockerfileを用いたクラウド環境への継続的デプロイ (CI/CD)

## 3. コア機能 (Core Features)

1. **部品・取引先マスター管理**
   部品の単価・リードタイム・単位などの基本情報や、取引先(Supplier)の連絡先情報の登録と管理。
2. **発注管理 (Order Management)**
   直感的なUIを通じた部品発注。一部入庫済みの発注データは修正を自動ブロックするなど、厳格な状態制御を実装。
3. **入庫管理 (Receiving & Logging)**
   バーコード/QRリーダーでの入力を想定した迅速な入庫処理。在庫の即時加算と、トランザクションによる入庫履歴（Receipt Log）の確実な記録。
4. **在庫・廃棄管理 (Inventory & Disposal)**
   リアルタイムな在庫追跡。欠陥部品や陳腐化した在庫に対する「廃棄処理」と理由のロギング。
5. **計上および精算書自動生成 (Accounting & PDF Export)**
   指定期間内の入庫履歴を取引先別に集計。さらに、日本語フォント(`IPAexGothic`)を搭載した高精度な**PDF精算書**をバックエンドで動的に描画し、一括でZIPファイルとしてダウンロード可能。

## 4. 技術的な注力ポイント

1. **データ不整合の徹底的な防止とマイグレーション**
   発注～入庫～計上に至る一連のプロセスで Spring のトランザクション管理を活用。さらに、過去の「取引先未指定」のレガシーデータを安全なダミー取引先(`AA01`)へ自動移行するバッチ処理(`DataMigrationRunner`)を実装。
2. **セキュアで堅牢なPDFエンジン設計**
   Spring BootのFAT JAR実行環境における静的ファイル参照問題を解決するため、TTFフォントリソースを起動時に一時領域(Temp)へ物理・展開してからPDF描画エンジンへ渡すことで、環境非依存の安定稼働を実現。
3. **動的でユーザーフレンドリーなUI制御**
   発注ステータス(PENDING)や入庫数量(Quantity)などの条件付きロジックを Thymeleaf 上で計算し、ユーザーが誤った操作を行えないように編集ボタンや入力領域を動的に制御。
4. **環境依存情報の分離**
   DBの接続情報やデプロイ設定などの機密情報を環境変数（Environment Variables）へと分離し、GitHub上への情報露出を防ぐセキュアな設計。
